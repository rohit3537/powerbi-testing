- Adding Retail Item Rank, FeatureCode, SeasonCode slicers to all report pages with Sales_Inv visuals
- Saved workbook as .pbip to enable programmatic report editing

## Decisions made
- Created 3 dedicated calculated lookup tables (`_Retail Item Rank`, `_FeatureCode`, `_SeasonCode`) in the semantic model
- Each lookup uses `DISTINCT(UNION(DISTINCT(Sales_Inv[col]), DISTINCT('DC Inventory'[col])))` to capture all values from both fact tables
- Created 6 relationships (Sales_Inv → each lookup, DC Inventory → each lookup)
- All 19 slicers across 7 pages now point at these lookup tables instead of Sales_Inv directly
- Did NOT add columns to Item Info — those columns are row-level attributes, not product-level, so they came through as blanks after dedup

## Why lookup tables were needed
- Retail Item Rank, FeatureCode, SeasonCode only exist on Sales_Inv and DC Inventory (fact tables)
- No shared dimension existed for these columns
- Slicers sourcing from Sales_Inv showed blank on pages with visuals from other tables (no relationship path)
- Copy-paste of slicers from Overview had the same problem
